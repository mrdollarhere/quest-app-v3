# DNTRNG™ Registry Bridge Protocol - Action Registry (v19.2.4)

This document serves as the forensic checklist for all supported handshakes between the Next.js frontend and the Google Apps Script (GAS) backend.

## 1. GET Actions (doGet)
Used for retrieving data from the Google Sheet registry. Parameters are passed via URL search queries.

- [ ] **`getVersion`**: Returns the current script version (v19.2.4) to ensure frontend compatibility.
- [ ] **`getBugReports`**: Retrieves the site-wide issue registry from the `BugReports` tab.
- [ ] **`getActivity`**: Retrieves the unified forensic telemetry stream from `System_Activity`.
- [ ] **`login`**: Verifies email and password against the `Users` tab. Returns identity metadata (minus password).
- [ ] **`getUsers`**: Retrieves the full student registry for administrative oversight.
- [ ] **`getTests`**: Fetches the assessment module library, enriched with real-time question counts from dynamic sheets.
- [ ] **`getResponses`**: Retrieves mission submission logs. Supports an `email` filter for identity-targeted retrieval.
- [ ] **`getSettings`**: Fetches platform calibration parameters as a key-value dictionary.
- [ ] **`getQuestions`**: Retrieves the full question set for a specific module ID (includes correct answers).

## 2. POST Actions (doPost)
Used for committing, updating, or purging data in the registry. Payloads are transmitted as JSON bodies.

- [ ] **`saveBugReport`**: Records student-submitted technical discrepancies with auto-captured context.
- [ ] **`updateBugStatus`**: Updates the triage status (`new`, `reviewing`, `resolved`) and internal admin notes.
- [ ] **`logActivity` / `logEvent`**: Records telemetry pulses, navigation events, and IP/Device signatures.
- [ ] **`submitResponse`**: Commits finalized assessment results, including scores, durations, and anti-cheat metadata.
- [ ] **`deleteResponse`**: Permanently purges a specific submission log from the `Responses` tab.
- [ ] **`saveTest`**: Performs an upsert on test metadata and forensicallly initializes the corresponding question tab.
- [ ] **`deleteTest`**: Purges a test module from the library and deletes its associated question sheet.
- [ ] **`saveUser`**: Single student provisioning or role calibration.
- [ ] **`saveUsers`**: High-velocity batch provisioning for classrooms (array of identity nodes).
- [ ] **`deleteUser`**: Purges a specific student identity from the registry.
- [ ] **`saveQuestion`**: Single question node calibration (upsert logic).
- [ ] **`saveQuestions`**: Complete registry overwrite for a module's question set.
- [ ] **`saveSetting`**: Calibrates a global system preference (e.g., `join_mode`, `theme_color`).

## 3. Security Protocol
All actions require the `apiKey` parameter/field to match the `INTERNAL_API_KEY` defined in the GAS script and environment variables. Failure to provide a valid key results in a `401 Unauthorized` pulse.
